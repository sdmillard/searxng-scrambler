{% set _routing = prefs.use_tor if prefs.use_tor in ('tor', 'tor_fallback', 'direct') else ('tor' if prefs.use_tor else 'direct') %}

When using Tor, your real IP is never sent to any instance. DNS also resolves through Tor. Tor with fallback falls back to direct if Tor is unreachable or times out (configure the timeout below) — captchas and rate-limits are per-instance problems, not Tor failures, so they never trigger a fallback. Tor only never falls back regardless of timeout.

Privacy trade-off: When above 0, Scrambler connects to check.torproject.org through that many Tor circuits at startup — before you search anything — so the circuits are already built when you need them. First-search latency drops significantly. Setting it to match your instances-per-search means every parallel request gets a pre-warmed circuit.

The downside: Tor makes outbound connections at rest. An observer watching your Tor traffic can see activity even when you haven't searched. Leave at 0 unless first-search speed is a priority.

Leave blank to use your system resolver. Only applies when Tor is off.

How long to wait for a Tor response. Tor with fallback: when set above 0, falls back to direct after this many seconds or if Tor is unreachable; when set to 0, only falls back if Tor can't connect at all — slow responses don't trigger a fallback. Tor only: request is abandoned after this time with no fallback (0 = wait indefinitely). Applies to searches, thumbnails, geocoding, and all other Tor traffic. Default 12 s.

Last-resort fallback when all SearXNG instances fail to return results for an engine: Scrambler queries that engine directly using the same routing as searches (Tor → direct if tor_fallback). How many additional instances to try when a search fails. Applies in two situations: (1) if all initial concurrent instances fail, this many more are tried one at a time with the full engine list; (2) after the first success, this many more instances are tried for any engines that returned no results, sending only the missing engines each time. Set to 0 to disable all fallback. Maximum is capped to your instance count ({{ instance_count }}).

Total time allowed for the engine failover phase. With Tor circuits taking 8–13 s each, 60 s allows ~4–7 attempts. Set to 0 for no time limit — failover runs until all engines are accounted for or the instance list is exhausted. Less relevant in live mode where results stream in as they arrive. Default 60 s.
